# Contributing

Thanks for considering contributing to this project! Contributions that improve documentation, fix bugs, or add small features are welcome.

Getting started
- Fork the repository and create a branch named `feat/<short-description>` or `fix/<short-description>`.
- Keep changes focused and limited to a single concern.

Code style
- Follow existing TypeScript + Angular style in `src/`.
- Run tests and linters before opening a PR.

Commit messages
- Use clear, present-tense messages. Example: `fix: correct sprite URL` or `feat: add team management`.

Pull requests
- Open a PR against `main` with a concise description of the change and why it is needed.
- Link relevant issues and include screenshots for UI changes.

Tests and checks
- Ensure `npm test` passes and add unit tests when appropriate.
